# Конвенции именования

## Имена

- Файлы модулей (не компоненты): `camelCase.ts`
- Файлы компонентов React: `PascalCase.tsx`
- Стили для компонентов: `{Component}.css.ts` (web), `{Component}.styles.ts` (mobile)
- Общие стили: `{styles}.css.ts` (web), `{styles}.styles.ts` (mobile)

## Файлы рядом с фичей или модулем

- **Константы**, которые относятся к одной директории и импортируются из **нескольких** файлов в ней: вынеси в `constants.ts` в этой же директории
- **Типы**, общие для нескольких файлов в директории: `types.ts` в этой же директории

## Утилиты и хуки

- Директория **`utils/`** — по одному файлу на утилиту; имя файла совпадает с именем экспортируемой функции или основного символа (например, `formatRelativeTime.ts` для `formatRelativeTime`)
- Директория **`hooks/`** — по одному файлу на хук; имя файла совпадает с именем хука (например, `useOnlineStatus.ts` для `useOnlineStatus`)

Не склеивай несколько несвязанных хуков или утилит в один файл внутри `hooks/` и `utils/`.
