# Документация проекта

Эта директория содержит архитектурные правила, соглашения по коду и инструкции для AI-ассистентов (Cursor и др.).

## Содержимое

### rules/

Правила, которым должен следовать код в монорепе:

- **architecture.mdc** — границы пакетов, зависимости между слоями, структура модулей, иерархия компонентов
- **code-style.mdc** — именование, стиль кода, экспорты
- **styling-rules.mdc** — стилизация в `packages/ui` (Vanilla Extract для web, StyleSheet для mobile)

### skills/

Пошаговые инструкции для создания новых сущностей:

- **add-feature** — планирование и реализация фичи через core → ui → web → mobile
- **create-core-module** — создание модуля в `packages/core` (store, API, hooks, lib)
- **create-ui-component** — создание кросс-платформенного компонента в `packages/ui`

### agents/

Описания специализированных агентов:

- **researcher** — исследование кодовой базы, поиск паттернов и связей
- **reviewer** — код-ревью, проверка архитектуры и качества
- **tester** — написание и запуск тестов

## Использование

- **Без Cursor** — читайте как документацию по архитектуре и стилю проекта
- **С Cursor** — скопируйте нужное в `.cursor/` (rules, skills, agents) для автоматического применения правил или подключайте через `@docs/` при необходимости
