# Документация проекта

Эта директория содержит архитектурные правила, соглашения по коду и инструкции для AI-ассистентов (Cursor и др.).

## Содержимое

### rules/

Правила, которым должен следовать код в монорепе:

- **architecture.md** — границы пакетов, зависимости между слоями, структура модулей, иерархия компонентов
- **code-style.md** — стиль функций, экспорты, обработчики событий
- **naming-conventions.md** — имена файлов, `constants.ts` / `types.ts`, каталоги `utils/` и `hooks/`
- **styling-rules.md** — стилизация в `packages/ui` (Vanilla Extract для web, StyleSheet для mobile)

### skills/

Пошаговые инструкции для создания новых сущностей:

- **create-ui-component** — создание кросс-платформенного компонента в `packages/ui`

### agents/

Описания специализированных агентов:

- **researcher** — исследование кодовой базы, поиск паттернов и связей
- **reviewer** — код-ревью, проверка архитектуры и качества
- **tester** — написание и запуск тестов

## Синхронизация с Cursor

Англоязычные зеркала лежат в **`.cursor/`** (`rules/*.mdc`, те же разделы `agents/`, `skills/`). При изменении `docs/` обновляй `.cursor/`, чтобы содержание совпадало (язык в `.cursor/` — английский).

## Использование

- **Без Cursor** — читайте как документацию по архитектуре и стилю проекта
- **С Cursor** — правила подхватываются из `.cursor/rules/`; при необходимости подключайте фрагменты через `@docs/`
